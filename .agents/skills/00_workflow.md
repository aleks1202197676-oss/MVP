# Workflow Skill: Preflight → Implement

## Purpose
Единый операционный протокол для любых PR.

## Steps
1. **Preflight**
   - Найти entrypoint/кодовый путь.
   - Зафиксировать точный список файлов для изменения.
   - Описать top-5 failure modes + mitigation.
   - Указать acceptance и команды проверки.
2. **Implement**
   - Вносить минимальные изменения в согласованном scope.
   - Сохранять детерминизм (порядок, формат, отсутствие плавающих значений).
3. **Verify**
   - Выполнить проверки из acceptance.
   - Проверить, что изменены только ожидаемые файлы.
4. **Deliver**
   - Краткий summary + результаты проверок.

## Mandatory quality rules
- Fail-fast диагностика вместо длинных логов.
- Никаких placeholder-путей/значений.
- Никаких необоснованных расширений scope.
