# Handoff: проект MVP

Этот репозиторий содержит MVP-пайплайн, который собирает артефакты проекта и формирует единый `latest_bundle.zip` для передачи состояния и контракта ИИ. Основная логика запуска находится в `pipelines`, а параметры сборки и включения файлов задаются в `config/hub.yaml`.

Цель проекта — поддерживать воспроизводимую сборку и понятную точку передачи контекста между итерациями разработки, чтобы изменения можно было быстро подхватывать в новой сессии без потери актуального состояния.

## Текущий статус

- Настроен CI, который запускает пайплайн и проверяет наличие `latest_bundle.zip`.
- Определён список обязательных файлов внутри бандла.
- Добавлен `memory_pack` для handoff-заметок и истории чатов.
- `memory_pack` включён в правила сборки бандла.

## Что делать дальше

- Поддерживать `memory_pack/handoff.md` в актуальном состоянии после каждого значимого изменения.
- Сохранять промежуточные заметки/контекст в `memory_pack/chats/`.
- Перед открытием PR запускать локальную проверку сборки бандла.
- Следить, чтобы обязательные bundle-entries в CI соответствовали реальным требованиям.
- При изменении структуры проекта обновлять `bundle.include_globs` в `config/hub.yaml`.

## Как собрать/обновить `latest_bundle.zip`

```bash
python -m pipelines.run_all
```

После выполнения архив будет обновлён по пути: `data/ai_contract/latest/latest_bundle.zip`.

## Где скачать бандл

С релиза `latest-bundle`: `https://github.com/<owner>/<repo>/releases/tag/latest-bundle`.

## Важные правила по веткам/PR

- Рабочую ветку создавать от актуального `main`.
- Pull Request открывать с `base=main`.
- Перед merge убедиться, что CI-checks зелёные и бандл содержит обязательные файлы.
