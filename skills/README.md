# Skills Repository Memory

`skills/` — это процессная память репозитория: короткие шаблоны задач и чеклисты, которые уменьшают количество ошибок в PR.

## Scope
- Только процесс и качество поставки.
- Без продуктовой логики, без данных и без дублирования контрактов.

## Правила оформления
- Формат: только Markdown (`.md`).
- Каждый шаблон должен содержать: `Goal`, `Hard rules`, `Preflight`, `Implementation`, `Acceptance`, `Checks`.
- Чеклисты должны быть короткими, проверяемыми и привязанными к командам.

## Правило обновления
- Каждый PR должен добавить **минимум 1 улучшение** в `skills/**` **или** ссылку на конкретный шаблон/чеклист из `skills/**` в описании PR.

## Anti-bloat policy
Чтобы `skills/` не превратился в свалку:
1. Один сценарий — один шаблон.
2. При появлении дубликатов старый файл удаляется или становится redirect-файлом на новый.
3. Любой новый шаблон должен объяснять, какую повторяющуюся ошибку он предотвращает.
4. Не добавлять длинные теоретические объяснения — только операционные шаги.

## Ownership
- Owner: команда process/gatekeeper.
- Ревью: изменения в `skills/**` желательно проверять вместе с CI/gatekeeper изменениями.
