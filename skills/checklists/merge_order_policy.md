# Checklist: Merge Order Policy

## Базовый порядок
1. Сначала PR с инфраструктурой/гейтами (обычно `PR-G*`).
2. Затем функциональные PR (`PR-F*`, `PR-A*`), которые зависят от новых правил.
3. В конце — docs-only синхронизация (если не критична для блокеров).

## Правила
- Не мержить PR с конфликтующим scope одновременно.
- Если меняется CI/gatekeeper, сначала убедиться, что downstream ветки ребейзнуты.
- При срочных фикcах hot path допускается исключение, но с явным post-merge follow-up.
