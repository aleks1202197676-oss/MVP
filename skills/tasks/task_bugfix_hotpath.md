# TASK Template: Bugfix (Hot Path)

## Goal
Исправить дефект в критичном пути без расширения scope.

## Hard rules
- Не менять архитектуру вне hot path.
- Не добавлять новые зависимости.
- Сохранить обратную совместимость входов/выходов.

## Preflight
1. Минимальная воспроизводимость (1 команда / 1 сценарий).
2. Что сломано сейчас (ожидаемое vs фактическое).
3. Где оптимальная точка фикса.
4. Риски регрессии.

## Implementation
- Самый короткий путь к fix.
- Добавить guard/validation там, где ошибка возникает.
- Сообщение об ошибке должно быть actionable.

## Acceptance
- Репродьюсер до фикса падает, после фикса проходит.
- Нет побочных изменений в соседних модулях.

## Checks
- Команда репродьюсера.
- Целевой smoke/regression check.
