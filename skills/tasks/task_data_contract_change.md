# TASK Template: Data Contract Change

## Goal
Изменить контракт данных контролируемо и прозрачно для downstream.

## Hard rules
- Явно указать breaking/non-breaking.
- Обновить примеры входов/выходов.
- Не смешивать контрактные и инфраструктурные изменения в одном PR.

## Preflight
1. Текущая версия контракта и потребители.
2. Матрица влияния (кто сломается и почему).
3. План миграции и совместимости.

## Implementation
- Изменения контракта + синхронное обновление документации.
- Явные валидации на границах.

## Acceptance
- Потребители либо совместимы, либо имеют явный migration path.
- Документация отражает фактический контракт.

## Checks
- Проверка схем/примеров.
- Smoke-run пайплайнов-потребителей (если применимо).
