# TASK Template: Docs-only Change

## Goal
Обновить документацию без изменения исполняемого кода.

## Hard rules
- Не менять `.py`, workflow и данные.
- Сохранять консистентность терминов и ссылок.

## Preflight
1. Какие документы источник истины.
2. Какие секции устарели.
3. Точный список файлов docs для правки.

## Implementation
- Короткие, проверяемые формулировки.
- Примеры команд должны запускаться как написано.

## Acceptance
- Нет изменений вне docs/skills (или явно согласованных файлов).
- Нет битых внутренних ссылок.

## Checks
- `git diff --name-only`
- Быстрый проход по ссылкам/командам вручную.
