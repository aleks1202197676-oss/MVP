# TASK Template: PRE-FLIGHT + IMPLEMENT

## Goal
Кратко: что должно измениться и какой измеримый результат нужен.

## Hard rules
- Preflight обязателен до изменений.
- Ограничение по scope (какие папки можно/нельзя трогать).
- Без новых зависимостей (если явно не разрешено).

## Preflight (STOP после отчёта)
1. Где находится текущая логика (entrypoints + функции).
2. Источники недетерминизма/рисков.
3. Точный список файлов для изменения/создания.
4. Acceptance criteria и команды проверки.
5. Top-5 failure modes + mitigation.

## Implementation
- Минимальный патч по согласованному scope.
- Явная диагностика ошибок.
- Без изменения внешних интерфейсов, если это не требуется задачей.

## Acceptance
- Все критерии из preflight выполнены.
- Логи понятны: причина и путь исправления.

## Checks
- `git diff --name-only`
- Целевые тесты/команды из задачи.
- Повторный прогон для проверки детерминизма (если применимо).
